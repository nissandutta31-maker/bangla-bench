# BanglaBench

**A unified, native-speaker-maintained leaderboard for Bengali (বাংলা) LLM evaluation.**

Bengali has 250M+ speakers and almost no consolidated, up-to-date LLM evaluation. Academic Bengali benchmarks are scattered across individual papers, and general "multilingual" leaderboards usually score Bengali through machine translation — which measures the translation as much as the model. BanglaBench is one place to see how frontier proprietary models *and* Bangla-native models actually perform on Bengali tasks: scored directly in Bengali, closed-book, each model on its own, and refreshed as new models ship.

> **Status: v0 MVP.** Belebele reading comprehension (900 items) plus a native Bengali MCQ seed set. Full multi-model leaderboard runs when API keys are exported.

## Methodology

- **Task:** 4-way MCQ. The model receives the **passage** (when present), the question, and four options, and answers with a single letter (A–D).
- **Scoring:** exact-letter match · **temperature 0** · **closed-book** (no tools, no retrieval).
- **Per-model independence:** every model is scored and ranked on its own. Scores are never averaged or failed-over across providers.
- **Per-model `max_tokens`:** 2048 for reasoning/frontier models; 32 for non-reasoning models (V3, Llama 3.3 NIM).
- **Answer parsing:** robust to verbose/markdown output from frontier models (see `parse_answer` in the runner).
- **Native-speaker maintained:** answer keys and item fluency are reviewed rather than trusted blindly from machine translation.

## Quickstart — reproduce the leaderboard

```bash
pip install -r requirements.txt

# Export keys for whichever models you want ranked (see model table above).
export DEEPSEEK_API_KEY=...
export OPENAI_API_KEY=...
export ANTHROPIC_API_KEY=...
export GEMINI_API_KEY=...
export NVIDIA_API_KEY=...
export HF_TOKEN=...          # Bangla-native models via Hugging Face Inference
# export HF_API_BASE=...     # optional: self-hosted TGI/vLLM endpoint

# Full Belebele split (900 items) — primary v0 benchmark
python3 run_leaderboard.py

# Quick smoke run
python3 run_leaderboard.py belebele_ben_sample.jsonl

# Native Bengali MCQ seed benchmark
python3 run_leaderboard.py bangla_mcq_native.jsonl

# All registered benchmarks + cross-benchmark summary
python3 run_leaderboard.py --all-benchmarks
```

Outputs:

- `leaderboard.md` / `leaderboard.csv` — Belebele full split (default)
- `leaderboard_<benchmark>.md` — per-benchmark files for other datasets
- `leaderboard_summary.md` — model × benchmark matrix when running multiple datasets

Offline tests (no API calls):

```bash
python3 test_smoke.py
```

## The runner (infrastructure)

`bangla_bench_runner.py` is the underlying evaluation engine: provider routing, retry with exponential backoff, cross-provider failover for reliability, answer parsing, and a per-attempt CSV audit log.

```bash
python3 bangla_bench_runner.py prompt "প্রশ্ন: বাংলাদেশের রাজধানী কোনটি? A. ঢাকা B. খুলনা C. চট্টগ্রাম D. সিলেট"
python3 bangla_bench_runner.py eval sample_items.jsonl -o results.jsonl
```

**Note:** the `eval` subcommand uses cross-provider failover, which *blends* providers and is **not** how the leaderboard is scored. For per-model standings, always use `run_leaderboard.py`.

## Input format (Belebele / BanglaBench JSONL)

One JSON object per line. The normalizer accepts common field-name variants:

- **id:** `item_id` · `id` · `task_id`
- **passage:** `flores_passage` · `passage` · `context` (may be empty for native MCQ)
- **question:** `question` · `question_text`
- **choices:** `mc_answer1`…`mc_answer4` *or* a `choices` list
- **gold:** `correct_answer_num` (1–4 → A–D) · `answer` · `correct_answer`
